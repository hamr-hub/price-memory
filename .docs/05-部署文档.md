# Price Memory 部署文档

## 部署概述

Price Memory 支持多种部署方式，包括本地开发部署、Docker容器化部署和Kubernetes集群部署。本文档详细介绍各种部署方式的配置和操作步骤。

## 环境要求

### 基础环境
- **操作系统**: Linux (推荐 Ubuntu 20.04+) / macOS / Windows
- **Docker**: >= 20.10.0
- **Docker Compose**: >= 2.0.0
- **Kubernetes**: >= 1.20 (集群部署)

### 资源要求
- **CPU**: 最低 2 核，推荐 4 核
- **内存**: 最低 4GB，推荐 8GB
- **存储**: 最低 20GB，推荐 50GB
- **网络**: 稳定的互联网连接

## 本地开发部署

### 1. 环境准备
```bash
# 克隆项目
git clone https://github.com/your-username/price-memory.git
cd price-memory

# 创建环境配置文件
cp .env.example .env
```

### 2. 配置环境变量
编辑 `.env` 文件：
```env
# 数据库配置
DATABASE_URL=postgresql://postgres:password@localhost:5432/price_memory
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-supabase-anon-key

# Redis配置
REDIS_URL=redis://localhost:6379

# API配置
API_HOST=0.0.0.0
API_PORT=8000
SECRET_KEY=your-secret-key-here

# 前端配置
VITE_API_URL=http://localhost:8000
VITE_GRAPHQL_URL=https://your-graphql-url/graphql
```

### 3. 启动服务
```bash
# 启动数据库和Redis
docker-compose up -d postgres redis

# 启动后端服务
cd spider
uv run python main.py

# 启动前端服务
cd admin
npm run dev
```

## Docker 容器化部署

### 1. 构建镜像

#### 后端镜像构建
```dockerfile
# spider/Dockerfile
FROM python:3.14-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    curl \
    && rm -rf /var/lib/apt/lists/*

# 安装uv
RUN pip install uv

# 复制依赖文件
COPY pyproject.toml uv.lock ./

# 安装Python依赖
RUN uv sync --frozen

# 安装Playwright浏览器
RUN uv run playwright install --with-deps chromium

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["uv", "run", "python", "main.py"]
```

#### 前端镜像构建
```dockerfile
# admin/Dockerfile
FROM node:18-alpine AS builder

WORKDIR /app

# 复制依赖文件
COPY package*.json ./

# 安装依赖
RUN npm ci

# 复制源代码
COPY . .

# 构建应用
RUN npm run build

# 生产环境镜像
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制nginx配置
COPY nginx.conf /etc/nginx/nginx.conf

# 暴露端口
EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

### 2. Docker Compose 部署

创建 `docker-compose.yml` 文件：
```yaml
version: '3.8'

services:
  # 数据库服务
  postgres:
    image: postgres:15
    environment:
      POSTGRES_DB: price_memory
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./database/init.sql:/docker-entrypoint-initdb.d/init.sql
    ports:
      - "5432:5432"
    restart: unless-stopped

  # Redis缓存服务
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    restart: unless-stopped

  # 后端API服务
  api:
    build:
      context: ./spider
      dockerfile: Dockerfile
    environment:
      DATABASE_URL: postgresql://postgres:passworres:5432/price_memory
      REDIS_URL: redis://redis:6379
      API_HOST: 0.0.0.0
      API_PORT: 8000
    ports:
      - "8000:8000"
    depends_on:
      - postgres
      - redis
    restart: unless-stopped

  # 前端管理界面
  admin:
    build:
      context: ./admin
      dockerfile: Dockerfile
    environment:
      VITE_API_URL: http://localhost:8000
    ports:
      - "80:80"
    depends_on:
      - api
    restart: unless-stopped

volumes:
  postgres_data:
```

### 3. 启动服务
```bash
# 构建并启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## Kubernetes 集群部署

### 1. 创建命名空间
```yaml
# namespace.yaml
apiVersion: v1
kind: Namespace
metadata:
  name: price-memory
```

### 2. 配置 ConfigMap
```yaml
# configmap.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: price-memory-config
  namespace: price-memory
data:
  DATABASE_URL: "postgresql://postgres:password@postgres:5432/price_memory"
  REDIS_URL: "redis://redis:6379"
  API_HOST: "0.0.0.0"
  API_PORT: "8000"
```

### 3. 配置 Secret
```yaml
# secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: price-memory-secret
  namespace: price-memory
type: Opaque
data:
  SECRET_KEY: <base64-encoded-secret-key>
  SUPABASE_KEY: <base64-encoded-supabase-key>
```

### 4. 数据库部署
```yaml
# postgres.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: postgres
  namespace: price-memory
spec:
  replicas: 1
  selector:
    matchLabels:
      app: postgres
  template:
    metadata:
      labels:
        app: postgres
    spec:
      containers:
      - name: postgres
        image: postgres:15
        env:
        - name: POSTGRES_DB
          value: price_memory
        - name: POSTGRES_USER
          value: postgres
        - name: POSTGRES_PASSWORD
          value: password
        ports:
        - containerPort: 5432
        volumeMounts:
        - name: postgres-storage
          mountPath: /var/lib/postgresql/data
      volumes:
      - name: postgres-storage
        persistentVolumeClaim:
          claimName: postgres-pvc

---
apiVersion: v1
kind: Service
metadata:
  name: postgres
  namespace: price-memory
spec:
  selector:
    app: postgres
  ports:
  - port: 5432
    targetPort: 5432

---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: postgres-pvc
  namespace: price-memory
spec:
  accessModes:
  - ReadWriteOnce
  resources:
    requests:
      storage: 10Gi
```

### 6. 后端API部署
```yaml
# api.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: api
  namespace: price-memory
spec:
  replicas: 3
  selector:
    matchLabels:
      app: api
  template:
    metadata:
      labels:
        app: api
    spec:
      containers:
      - name: api
        image: price-memory/api:latest
        envFrom:
        - configMapRef:
            name: price-memory-config
        - secretRef:
            name: price-memory-secret
        ports:
        - containerPort: 8000
        livenessProbe:
          httpGet:
            path: /health
            port: 8000
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 8000
          initialDelaySeconds: 5
          periodSeconds: 5

---
apiVersion: v1
kind: Service
metadata:
  name: api
  namespace: price-memory
spec:
  selector:
    app: api
  ports:
  - port: 8000
    targetPort: 8000
  type: ClusterIP
```

### 8. Ingress 配置
```yaml
# ingress.yaml
apiVersion: networking.k8s.io/v1
kind: Ingress
metadata:
  name: price-memory-ingress
  namespace: price-memory
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    cert-manager.io/cluster-issuer: letsencrypt-prod
spec:
  tls:
  - hosts:
    - price-memory.example.com
    secretName: price-memory-tls
  rules:
  - host: price-memory.example.com
    http:
      paths:
      - path: /api
        pathType: Prefix
        backend:
          service:
            name: api
            port:
              number: 8000
      - path: /
        pathType: Prefix
        backend:
          service:
            name: admin
            port:
              number: 80
```

## 备份和恢复

### 1. 数据库备份
```bash
# 创建备份
kubectl exec -n price-memory postgres-pod -- pg_dump -U postgres price_memory > backup.sql

# 恢复备份
kubectl exec -i -n price-memory postgres-pod -- psql -U postgres price_memory < backup.sql
```

### 2. 自动化备份
```yaml
# CronJob备份任务
apiVersion: batch/v1
kind: CronJob
metadata:
  name: postgres-backup
  namespace: price-memory
spec:
  schedule: "0 2 * * *"  # 每天凌晨2点
  jobTemplate:
    spec:
      template:
        spec:
          containers:
          - name: backup
            image: postgres:15
            command:
            - /bin/bash
            - -c
            - pg_dump -h postgres -U postgres price_memory > /backup/backup-$(date +%Y%m%d).sql
            volumeMounts:
            - name: backup-storage
              mountPath: /backup
          volumes:
          - name: backup-storage
            persistentVolumeClaim:
              claimName: backup-pvc
          restartPolicy: OnFailure
```

## 故障排查

### 1. 常见问题
- **服务无法启动**: 检查配置文件和环境变量
- **数据库连接失败**: 验证数据库服务状态和连接字符串
- **内存不足**: 调整资源限制或增加节点
- **网络问题**: 检查Service和Ingress配置

### 2. 调试命令
```bash
# 查看Pod状态
kubectl describe pod <pod-name> -n price-memory

# 查看日志
kubectl logs <pod-name> -n price-memory

# 进入容器调试
kubectl exec -it <pod-name> -n price-memory -- /bin/bash

# 端口转发测试
kubectl port-forward service/api 8000:8000 -n price-memory
```

### 3. 健康检查
```bash
# API健康检查
curl http://localhost:8000/health

# 数据库连接检查
kubectl exec -it postgres-pod -n price-memory -- psql -U postgres -c "SELECT 1"
```
